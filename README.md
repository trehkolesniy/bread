# bread

bread — локальная читалка блогов для macOS. Она объединяет публикации из RSS- и Atom-лент в единую хронологическую ленту, отмечает новые материалы, открывает полные статьи во встроенном режиме чтения и умеет переводить превью и текст статей на русский язык.

Эта публичная сборка поставляется полностью пустой. В ней нет списка блогов, истории чтения, сохранённых статей, кэша переводов, API-ключей или других пользовательских данных.

## Системные требования

- macOS 14 Sonoma или новее;
- Mac с процессором Apple Silicon: M1, M2, M3, M4 или новее;
- доступ в интернет для загрузки блогов, статей, изображений и переводов.

Node.js уже включён в приложение. Устанавливать Node, npm, Homebrew или браузерные расширения не нужно.

## Запуск из исходного кода

Для готовой macOS-сборки этот раздел можно пропустить.

1. Установите Node.js 20 или новее.
2. Клонируйте репозиторий:

```sh
git clone https://github.com/trehkolesniy/bread.git
cd bread
```

3. Запустите проверку и локальный сервер:

```sh
npm run check
npm start
```

4. Откройте `http://127.0.0.1:4173/`.

Зависимости npm приложению не требуются. Для сборки нативного приложения понадобится Xcode Command Line Tools; команда `scripts/build-macos-app.command` создаст `~/Applications/bread.app`.

## Установка

1. Скачайте и распакуйте `bread-2.0.0-macos-arm64.zip`.
2. Переместите `bread.app` в папку `/Applications`.
3. Откройте `bread.app`.

Приложение подписано локальной ad-hoc подписью, но не прошло нотаризацию через Apple Developer. При первом запуске macOS может попросить подтвердить доверие к загруженному приложению. Используйте команду **Открыть** в контекстном меню Finder или проверьте заблокированное приложение в разделе **Системные настройки → Конфиденциальность и безопасность**, только если вы скачали bread из источника, которому доверяете.

bread запускает приватный локальный сервер по адресу:

`http://127.0.0.1:4173/`

Сервер доступен только на вашем Mac и не открывается для других устройств в сети.

## Добавление блогов

bread читает только те источники, которые вы добавили самостоятельно.

1. Откройте bread.
2. Нажмите **Библиотека** в правом нижнем углу.
3. Раскройте раздел **Добавить блог**.
4. Введите название блога.
5. Укажите прямую ссылку на RSS- или Atom-ленту.
6. Нажмите **Добавить**.
7. Нажмите кнопку обновления в нижней панели, чтобы загрузить публикации.

Примеры адресов лент:

- `https://example.com/feed.xml`
- `https://example.com/rss`
- `https://example.com/atom.xml`

Обычная ссылка на главную страницу блога может не работать, если по этому адресу не возвращается RSS- или Atom-документ. bread не сканирует произвольные сайты и не загружает материалы с доменов, которые не были зарегистрированы как источники.

## Варианты перевода

### Без API-ключа

bread может использовать публичный сервис Google Translate для автоматического перевода на русский язык. Ключ не требуется, но качество и доступность такого перевода могут меняться.

### С API-ключом Gemini

Gemini обычно даёт более естественный перевод заголовков, кратких описаний и полного текста статей.

1. Откройте [Google AI Studio API Keys](https://aistudio.google.com/api-keys).
2. Войдите в аккаунт Google.
3. Создайте API-ключ Gemini.
4. Добавьте ключ одним из описанных ниже способов.
5. Полностью закройте bread и откройте его снова.

Условия использования, лимиты, стоимость и доступность моделей определяются Google и могут меняться. Перед подключением ключа проверьте актуальную информацию в Google AI Studio.

### Простая настройка

Запустите файл `Configure Gemini.command` из этого архива и вставьте API-ключ, когда появится запрос.

Помощник создаст файл:

`~/Library/Application Support/bread/reader/.env`

Файл будет доступен только текущему пользователю macOS.

### Ручная настройка

1. Один раз запустите bread, затем полностью закройте приложение.
2. В Finder выберите **Переход → Переход к папке…**
3. Откройте:

   `~/Library/Application Support/bread/reader`

4. Создайте обычный текстовый файл с именем `.env`.
5. Добавьте в него:

```dotenv
GEMINI_API_KEY=ваш_ключ_из_google_ai_studio
GEMINI_MODEL=gemini-3.1-flash-live-preview
```

6. Сохраните файл и снова откройте bread.

Не добавляйте ключ внутрь `bread.app`, в скачанный ZIP, публичный репозиторий, скриншот или сообщение в службу поддержки.

Чтобы позднее отключить Gemini, закройте bread и удалите строку `GEMINI_API_KEY` из файла `.env`. После этого bread вернётся к переводу через Google Translate без ключа.

## Чтение

- Публикации сортируются по дате: новые находятся сверху.
- Новые материалы отмечаются пометкой **НОВОЕ**.
- Нажмите на заголовок, чтобы открыть полную статью в панели справа.
- В режиме статьи сохраняются изображения, подписи, содержательные ссылки, списки, цитаты, аудио, видео и поддерживаемые встроенные материалы.
- Переключатель **Оригинал / Перевод** меняет исходный текст на русский перевод и обратно.
- Тонкие линии сверху независимо показывают прогресс прокрутки ленты и открытой статьи.
- Кнопка с галочкой отмечает публикацию прочитанной.
- Кнопка со звездой сохраняет материал.
- Кнопка **Всё прочитано** отмечает текущую ленту прочитанной.

Изображения и другие медиаматериалы показываются только внутри полной статьи. Короткие превью в ленте остаются текстовыми.

## Библиотека и поиск

Откройте **Библиотеку**, чтобы:

- искать публикации по заголовку, автору, блогу или описанию;
- показывать все, новые, непрочитанные или сохранённые материалы;
- просматривать зарегистрированные источники;
- добавлять новые RSS- и Atom-ленты;
- удалять источники.

Нижняя панель также позволяет менять размер текста, выбирать шрифт без засечек или с засечками, переключать светлую и тёмную тему, обновлять ленту и включать полноэкранный режим.

## Локальные файлы и приватность

Все изменяемые пользовательские данные хранятся в папке:

`~/Library/Application Support/bread/reader`

Основные файлы:

- `.env` — необязательный ключ Gemini и название модели;
- `data/sources.json` — добавленные источники;
- `data/reader-state.json` — публикации и история чтения;
- `data/translations.json` — кэш переводов превью;
- `data/article-translations.json` — кэш переводов полных статей.

bread обращается к интернету только при необходимости:

- к добавленным RSS- и Atom-источникам;
- к страницам статей и медиаматериалам, принадлежащим этим источникам;
- к сервисам перевода Google, если перевод включён.

Ключ Gemini читается локальным сервером и отправляется только в API Gemini компании Google при запросе перевода. Ключ не встраивается в приложение.

## Резервное копирование

Закройте bread и скопируйте в безопасное место папку:

`~/Library/Application Support/bread`

Восстановление этой папки вернёт источники, историю чтения и кэшированные переводы. Считайте резервную копию конфиденциальной, если внутри находится файл `.env`.

## Полный сброс

Закройте bread и переместите в Корзину папку:

`~/Library/Application Support/bread`

При следующем запуске приложение откроется с пустой библиотекой. Вместе с папкой будут удалены локальный API-ключ, список источников, история чтения и кэш переводов.

## Решение проблем

### Приложение не открывается

- Убедитесь, что используется Mac с Apple Silicon и macOS 14 или новее.
- Переместите приложение в папку `/Applications`.
- Если macOS блокирует ненотаризованное приложение, при первом запуске используйте команду **Открыть** в Finder.

### Порт 4173 уже занят

Закройте другие копии bread и локальные программы, использующие порт `4173`, затем снова откройте приложение.

### После добавления блога публикации не появились

- Проверьте, что сохранена прямая ссылка на RSS- или Atom-ленту.
- Откройте адрес ленты в браузере и убедитесь, что он возвращает XML.
- Повторно обновите ленту.
- Некоторые сайты могут временно отклонять автоматические запросы.

### Перевод не запускается

- Полностью закройте и снова откройте bread после изменения `.env`.
- Проверьте, что ключ записан в одну строку после `GEMINI_API_KEY=`.
- Убедитесь, что `.env` находится в папке Application Support, а не внутри приложения.
- Проверьте состояние ключа и текущие лимиты в Google AI Studio.
- Если Gemini недоступен, bread использует Google Translate.

### Полная статья отображается не целиком

bread извлекает редакционный материал и удаляет навигацию, формы, комментарии, рекламу и скрипты. Для сложных интерактивных статей может потребоваться ссылка **Открыть оригинал**.

## Безопасность

- Никогда не публикуйте файл `.env`.
- Не вставляйте API-ключ в GitHub Issues, формы аналитики или публичные обращения в поддержку.
- Скачивайте обновления только из источника, которому доверяете.
- В публичной сборке нет предустановленных блогов или персональных данных.
